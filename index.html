How to Use Faithlife Styles

This style guide is intended as a reference for all HTML, CSS, and JavaScript code in Faithlife content items. Some pages will require deviations, but this style guide should be followed as much as possible for the sake of consistency, efficiency, and maintainability.

Faithlifebase Unification updates unity

In order to unify the styling between the various lines of business, an addition to the main Faithlife Style Guide has been added. See full unification documentation

Content

To get started with Faithlife styles, create a new content item using the following template.

<main class="faithlifebase">
                      
                            </main>

All HTML should go inside the <main> element. Each category of HTML in this style guide has certain limitations regarding where it can go. Everything should be placed inside Section components except for other Standalone Components.

Styles

To connect your project to this styleguide be sure to include it in the external styles section in Content Admin. This will automatically add it in the <head> of your new page in its own <link> tag.

https://www.logos.com/styles/styleguides/faithlifebase.css

Benefits of Using a Style Guide

  • Saves time for designers
    • Prevents redundant/duplicate work
    • Suggests a library of components that designers can use in their mockups
  • Saves time for developers
    • The same component (e.g. Product) can be updated across hundreds of pages at the same time
    • Empowers marketers to create pages without needing to ask developers
  • Saves time for marketers
    • HTML can be copy/pasted directly from the style guide
    • Project management time is reduced as less design and development work are needed
  • Improves user experience
    • Ensures consistent visuals across Faithlife monolithic brand sites
    • Prevents buggy pages since Faithlife styles will undergo greater quality assurance than one-off code
